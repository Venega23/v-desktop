# Слой

Полноэкранная панель быстрых знаний для Windows. Вызывается поверх текущей работы, показывает заметки, чек-листы, ссылки и изображения, а затем исчезает по `Esc` или повторному нажатию горячей клавиши.

## Обычный запуск в Windows

Соберите установщик и portable-версию одной командой:

```powershell
npm ci
npm run build:win
```

Готовые файлы появятся в папке `dist`:

- `V-Setup-<version>-x64.exe` — обычная установка с ярлыками и автообновлением;
- `V.exe` — portable-запуск без установки.

После сборки Node.js пользователю больше не нужен. `ffmpeg`, необходимый для надёжного сохранения длительности аудиозаписей, входит в приложение. Локальная сборка не подписана сертификатом издателя, поэтому Windows SmartScreen может показать предупреждение; для публичного распространения EXE следует подписать code-signing сертификатом.

## Запуск из исходников

```powershell
npm ci
npm start
```

На Windows также можно открыть `Запустить Слой.cmd`: при первом запуске он проверит зависимости и установит их, а затем запустит приложение. Этот вариант требует Node.js LTS и интернет только для первоначальной установки.

## Проверка перед сборкой

```powershell
npm run check
npm run test:smoke
npm audit
```

`check` проверяет синтаксис и регрессии основных функций. `test:smoke` запускает Electron на короткое время, дожидается готового renderer-процесса и завершается с ненулевым кодом при ошибке первого запуска.

## Обновления через GitHub

Установленная версия автоматически проверяет публичные релизы `Venega23/v-desktop` после запуска и затем раз в четыре часа. Проверку также можно запустить вручную из меню значка V в системном трее.

Обновление скачивается во временный кэш, а не в рабочую папку приложения. Перед запуском NSIS проверяется SHA-512 из `latest.yml`; при ошибке установленная версия не заменяется. Окно показывает этап, общий прогресс, файл, объём, скорость и оставшееся время. Технический журнал находится в каталоге логов профиля приложения.

Новый релиз создаётся тегом `v<версия>`: GitHub Actions запускает проверки, собирает Windows-установщик, portable-файл, `latest.yml` и blockmap, а затем прикрепляет их к GitHub Release. Пошаговая памятка находится в `RELEASING.md`.

После запуска приложение остаётся в системном трее.

- `Ctrl + Alt + Space` — показать или скрыть Слой.
- `Esc` — закрыть меню, выйти из правки или скрыть окно.
- `Ctrl + K` — поиск.
- `Ctrl + N` — новая текстовая карточка.
- `Ctrl + V` — создать карточку из скриншота, ссылки или текста в буфере.
- `E` — переключить «Обзор / Править».

## Что уже работает

- полноэкранное окно на мониторе, где находится курсор;
- системный трей и глобальная горячая клавиша;
- готовый шаблон «Новая работа»;
- текст, чек-листы, ссылки и изображения;
- независимые рабочие и личные пространства;
- добавление, редактирование, удаление с отменой, перестановка и изменение размеров карточек;
- режимы «Доска», «Галерея» и «Список» с отдельными настройками для каждого пространства;
- сортировка вручную, по новизне или названию;
- групповое выделение карточек и назначение общего размера S/M/W/L;
- вставка скриншота, текста и ссылки напрямую через `Ctrl+V`;
- ссылка у изображения: кнопка `↗` или двойной клик по картинке открывают её в системном браузере;
- выбор отображения изображения: целиком или с заполнением карточки;
- явная запись разговора с микрофона и локальное сохранение аудио;
- опциональная запись звука Windows вместе с микрофоном;
- отдельное сохранение дорожек микрофона и звука компьютера;
- защищённое подключение xAI или Groq Speech-to-Text;
- потоковая расшифровка и необязательные подсказки ответа во время разговора;
- крупные развёрнутые realtime-подсказки с двумя готовыми вариантами ответа, опорными фактами и следующим вопросом;
- карточка разговора создаётся в пространстве, где запись была начата, даже если затем открыть другое;
- краткая выжимка, ключевые пункты, решения, задачи и открытые вопросы;
- во время длинного разговора один набор live-карточек обновляется и повторно структурируется без создания карточки на каждую реплику;
- полная стенограмма свёрнута по умолчанию, а основная карточка показывает темы, решения, задачи и шпаргалки;
- длинные встречи обрабатываются частями: все важные пункты объединяются без лимита в 6 или 12 строк;
- выделенный текст в режиме правки можно форматировать: шрифт, размер, цвет, фон, списки, блоки и значки;
- автоматический черновик итогов, задач, людей, терминов и вопросов после записи;
- поиск по содержимому;
- режим приватности;
- локальное автосохранение в профиле приложения;
- reduced motion и режим высокой контрастности.

Изображения и аудио сохраняются отдельными файлами в профиле приложения. Метаданные карточек пока находятся в локальном browser storage; приложение сообщает об ошибке, если хранилище переполнено. Для крупных архивов встреч метаданные в дальнейшем стоит перенести в SQLite.

Кнопка «Включить запись» после запуска меняется на «Выключить запись» и остаётся единственным обычным способом остановить текущий отрезок. Escape, сворачивание и потеря фокуса только скрывают окно — запись продолжается. Микрофон записывается после явного подтверждения; звук компьютера по умолчанию выключен и при необходимости подключается отдельно. Перед записью разговора необходимо получить согласие участников. Если распознавание речи недоступно, аудио всё равно сохранится, а карточка будет ожидать подключения облачной транскрипции.

## Grok и распознавание речи

Откройте кнопку `✦` внизу sidebar, добавьте API-ключ xAI или Groq и выберите источники записи. Ключ шифруется через защищённое хранилище Windows и не передаётся в renderer/localStorage. xAI поддерживает настоящий потоковый STT; Groq расшифровывает независимые WAV-фрагменты во время записи, а после остановки обрабатывает полное аудио через Whisper Large V3 Turbo. Затем выбранная AI-модель создаёт черновик конспекта.

По умолчанию приложение пытается записывать две независимые дорожки: микрофон и звук компьютера. В панели записи всегда видно, какие источники действительно подключены. `Завершить`, повторный клик по кнопке записи или `Esc` останавливают сессию; локальное сохранение не зависит от доступности Grok.

Ключи `xai-…` отправляются только в xAI, а ключи `gsk_…` — только в Groq.

Sidebar меняется перетягиванием за правую границу от 64 до 520 px. При минимальной ширине интерфейс автоматически переходит в режим одних иконок. `Home` устанавливает минимальную ширину, `End` — максимальную, двойной клик возвращает стандартную.
